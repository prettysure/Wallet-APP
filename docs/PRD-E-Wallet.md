**文档版本：** 1.0  
**更新日期：** 2025-03  
**交付对象：** 开发团队

---

## 一、产品概述

### 1.1 产品定位

电子钱包（E-Wallet）是一款面向个人用户的电子钱包应用，用于**充值、缴费、提现**。产品不涉及资产投资或加密货币，仅作为资金流转与账单支付工具。

### 1.2 核心价值

- 用户可将资金从其他资金来源**充值**至钱包
- 用户可使用钱包**支付账单**（如水电、网络、燃气等）
- 用户可将钱包余额**提现**至其他账户或资金来源

### 1.3 技术形态

- 前端：单页应用（SPA），支持部署至 Vercel / Netlify 等静态托管
- 当前为演示版本，认证与数据为前端内存/本地存储，无后端服务

| 角色     | 说明 |
|----------|------|

## 三、功能需求

### 3.1 注册（Register）

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| 注册入口 | 提供注册页，路径 `/register` | 可从登录页「创建账号」进入 |
| 必填信息 | 姓名、邮箱、密码、确认密码 | 均为必填，前端校验 |
| 邮箱格式 | 仅接受格式 `xxxx@xxx.com` | 其他格式（如 `.org`、无 `.com`）提示：「请输入有效邮箱（格式：xxxx@xxx.com）」 |
| 密码规则 | 不少于 6 位 | 不足时提示错误 |
| 确认密码 | 与密码一致 | 不一致时提示「两次密码不一致」 |
| 重复邮箱 | 同一邮箱不可重复注册 | 提示：「该邮箱已注册，请登录或使用其他邮箱」 |
| 隐私与条款 | 用户需勾选同意隐私政策与服务条款 | 未勾选不可提交，提示：「请同意隐私政策和服务条款」 |
| 政策链接 | 提供「隐私政策」「服务条款」链接 | 可新开页查看 `/privacy`、`/terms` |
| 注册成功 | 注册成功后跳转至登录页 | 不自动登录，需用户再次登录 |

### 3.2 登录（Login）

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| 登录入口 | 提供登录页，路径 `/login` | 标题/副标题体现为「登录到电子钱包」 |
| 必填信息 | 邮箱、密码 | 均为必填 |
| 邮箱格式 | 同注册，仅 `xxxx@xxx.com` | 格式错误时提示同上 |
| 登录成功 | 校验通过后跳转至仪表盘（首页） | 路径 `/`，进入带底部 Tab 的主框架 |
| 登录失败 | 邮箱或密码错误 | 提示「邮箱或密码错误」 |

### 3.3 权限与路由

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| 未登录访问 | 访问 `/`、`/transactions`、`/account` 时未登录 | 重定向至 `/login` |
| 已登录访问 | 已登录时访问 `/login` 或 `/register` | 重定向至首页 `/` |
| 公开页 | `/login`、`/register`、`/privacy`、`/terms` 无需登录 | 直接可访问 |

### 3.4 主框架与底部导航（Tab）

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| Tab 数量与顺序 | 共 3 个 Tab：**仪表盘**、**交易记录**、**我的账户** | 从左到右固定顺序 |
| Tab 形态 | Tab 为页面内一块区域，非固定贴底 | 有边距、圆角、边框/阴影，随页面滚动 |
| 当前页高亮 | 当前路由对应 Tab 高亮（如主题色） | 与路由一致：`/` → 仪表盘，`/transactions` → 交易记录，`/account` → 我的账户 |
| 路由映射 | 点击 Tab 切换路由 | `/`、`/transactions`、`/account` |

### 3.5 仪表盘（Dashboard，首页 `/`）

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| 页头 | 展示「My E-Wallet」、问候语「Hello, {用户姓名}」、退出登录按钮 | 退出后回到登录页 |
| 可用余额 | 展示「Available balance」及金额（如 $12,450.78） | 金额格式千分位、两位小数 |
| 操作按钮 | 「Deposit」「Withdraw」与余额同一行或紧邻 | 点击为预留入口，可后续接真实充值/提现流程 |
| Quick pay（快捷缴费） | 独立区块，标题「Quick pay」 | 区块内为 biller 列表 |
| Biller 展示 | 在**一个方块区域**内展示所有 biller，每行约 2 个（或 4 列网格） | 每个 biller：上方为 **Logo**（可图为 logo 或首字母圆形），下方为 **Biller 名称**；整体紧凑 |
| Biller 交互 | 点击 biller 为预留入口 | 可后续接选择账单、输入金额、确认支付等 |
| 资金流水（Money flows） | 三个指标：**Total deposit**、**Total withdrawal**、**Total payment** | 以金额展示，可 mock 数据 |
| 余额表现（Balance performance） | 折线图展示一段时间内余额变化 | 见 3.6 |

### 3.6 余额表现（Balance performance）

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| 区块标题 | 标题为「Balance performance」 | 与「资金流水」等区块区分 |
| 时间范围切换 | 支持「Last 7 days」「Last 30 days」两种范围 | 以按钮或 Tab 切换，选中态高亮 |
| 图表 | 折线图，X 轴为日期，Y 轴为余额 | 悬停可显示具体日期与余额 |
| 说明文案 | 简短说明折线含义：存款使线上升，账单支付与提现使线下降 | 可放在图表上方或下方 |

### 3.7 交易记录（Transactions，`/transactions`）

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| 页面存在 | 有独立页面，标题如「Transactions history」 | 可从底部 Tab 进入 |
| 列表内容 | 展示交易列表（充值、提现、账单支付等） | 当前可为占位或 mock 列表，预留接口与字段 |

### 3.8 我的账户（My account，`/account`）

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| 展示信息 | 当前登录用户的姓名、邮箱 | 只读展示 |
| 退出登录 | 提供「Sign out」按钮 | 点击后清除登录态并跳转至 `/login` |

### 3.9 隐私政策与服务条款

| 需求项 | 描述 | 验收标准 |
|--------|------|----------|
| 隐私政策 | 独立页面 `/privacy`，描述数据收集与使用（与电子钱包用途一致） | 可从注册页链接打开 |
| 服务条款 | 独立页面 `/terms`，描述使用规范与免责等 | 可从注册页链接打开 |
| 返回 | 页面内可返回注册页或关闭 | 新开页打开时用户自行关闭或通过链接返回 |

## 四、非功能需求

### 4.1 UI/UX

- 整体为**深色主题**，与现有设计一致
- 移动端优先，最大宽度约 420px 居中
- 按钮、输入框具备 hover/active 等基础交互反馈
- 底部 Tab 为页面内卡片式，不固定贴屏底

### 4.2 兼容与部署

- 支持现代浏览器（Chrome、Safari、Edge 等）
- 构建产物为静态资源，可部署至任意静态托管
- 需配置 SPA 回退：所有路径返回 `index.html`，以便前端路由生效（如 Vercel/Netlify 已配置）

### 4.3 安全（当前版本）

- 密码不在前端持久化明文；当前为演示，实际需后端加密存储
- 邮箱格式与重复注册在前端校验，后端需再次校验并承担真实安全责任

## 五、路由与页面一览

| 路径 | 页面 | 需登录 |
|------|------|--------|
| `/` | 仪表盘（Dashboard） | 是 |
| `/transactions` | 交易记录（Transactions history） | 是 |
| `/account` | 我的账户（My account） | 是 |
| `/login` | 登录（Login） | 否 |
| `/register` | 注册（Register） | 否 |
| `/privacy` | 隐私政策（Privacy policy） | 否 |
| `/terms` | 服务条款（Terms of service） | 否 |

## 六、数据与接口（后续扩展）

当前为前端 Mock，后续开发团队可扩展：

- **用户**：注册/登录/登出、用户信息（姓名、邮箱等）
- **钱包**：可用余额、充值、提现、账单支付
- **Biller**：biller 列表（含名称、logo、编码等）
- **交易**：交易列表（类型、金额、时间、对方/描述等）
- **余额表现**：按 7 天/30 天查询余额快照或汇总
